# knowledge/ · 维度化知识库

按**认知维度**而不是时间/项目分类。

**默认维度**：
- `about-me/` — 关于你：身份、背景、价值观
- `tech/` — 技术理解（如果你做技术相关事）
- `playbooks/` — 可复用的 SOP（"怎么做某类事情"）
- `workflows/` — 方法论（"某件大事我一般按什么节奏走"）

**可选加**：
- `about-us/` — 关系（如果你希望 CC 知道你的合作者 / 家人）
- 任何你觉得有用的维度

**为什么维度化**：见 `blueprint/concepts.md` 第 3 节（读用途 + 去向两个判据）
