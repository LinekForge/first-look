# state-snapshots/ · 睡前便条

在重要工作节点写一份简短的便条，给下一次启动 CC 的你。让上下文跨 session 流动。

**命名**：`YYYYMMDD_HHMM[_context].md`
- `20260417_1145.md`
- `20260417_1145_ship_done.md`（加可选后缀说明）

**模板**：见下方。

**什么时候写**：
- Ship 完一个东西之后
- 压缩（/compact）之前
- 一天结束之前
- 有了重要 insight 时

**配合 SessionStart hook**（`hooks/bootstrap/session-start.sh`）会自动注入最近几个——你下次启动 CC 就能接上。

---

**模板**（可复制）：

```markdown
# 便条 · HH:MM {简短主题}

**写于**：{ISO 时间}
**对象**：下次醒来的我

---

## 当下

（现在做到哪了）

## 核心洞察

（今天想到的关键判断——可以是一两句）

## 下一步 candidates

（可能要做的事，不强制清单）

## 反思留档

（今天学到什么、踩了什么坑、为什么）
```

**写的节奏**：不要强迫每次都写——写了之后**真的会被下次用上**才值得。空写没意义。
